luckfox-交叉编译之bazel

发表于 2023-12-15 更新于 2023-12-19 分类于 luckfox ， bazel

最近opencv-mobile 对luckfox这款板子进行了适配优化，我对它也很有兴趣所以入手了一个luckfox开发板，包含300万像素摄像头，价格还算比较便宜。

不过官方提供的交叉编译链只有Linux以及windows的，而我却是Mac的，没办法直接调用，刚刚最近在使用bazel，并且它支持远程编译。

关于 luckfox 和 opencv-mobile

可以看看 opencv-mobile 现已支持 luckfox-pico MIPI CSI 摄像头和 rkaiq/rga 硬件加速

关于 Bazel

可以前往官网查看https://bazel.google.cn/，这里用到的主要是：https://github.com/bazelbuild/bazel-toolchains

开始准备

首先我们需要一个Linux系统，例如ubuntu，当然了，也可以使用windows的wsl，毕竟windows是最好的Linux桌面版本。

需要安装一下 docker 容器方便快速部署，或者可以考虑自己手动部署服务：bazel-buildfarm

为了方便，以及后续兼容，我们可以统一把工具链放到 /opt/toolchain， 我们需要把sdk里面的编译链放进来，编译链实际只需要这个目录的就可以：tools/linux/toolchain

例如将它放入到 ： /opt/toolchain/luckfox

然后就可以开始使用docker compose创建项目环境：

然后就可以执行 docker compose up -d(或者docker-compose up -d) 启动环境。

因为我们需要安装依赖，worker 可能会要几分钟时间才能注册到 server。可以通过 docker compose logs -f worker 查看日志。
